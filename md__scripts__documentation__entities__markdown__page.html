Each entity is defined in it's own scene. For example, NPC_Kobold_Spearman is defined in Scenes/NPC/NPC_Kobold_Spearman.tscn. These entities are the characters in the game, such as Kobolds or the Player themselves.

Owner Node

The top node in the tree, which is a KinematicBody2D.

On the player, this is where PlayerControls.cs is attached to match player input to movements of the character. NPC entities reference this node using Owner.<reference>.

AnimationPlayer

Control of animations is performed through the script attached to the AnimationPlayer, called AnimationTemplate.cs, which is generalized so that all entities (including the player) can use it.

Sprite

Used by the AnimationPlayer to modify the current sprite of the entity. The Light2D that's childed to this Node is what shows the 'Sight Radius' of the character (artistic flair).

Configuration

To configure the entity, create a child class of the ConfigurationTemplate.cs and name it after the entity it is configuring. This is where you will set values such as health, mana, faction, etc. Once this has been created, attach it to the entity in the entities scene to the <entity>/Configuration Node.

Interface

The EntityInterface.cs is where their current stats, or values, are stored. These would be the values set above in the Configuration, i.e. health, mana, etc. Use the getters and setters to access/set values. These functions are useful for Godot's Signals System as well.

Sensory

The ability of the entity to 'see' is controlled via the SensorController.cs. The CollisionShape2D radius determines the size of the Area2D that is used to see other entities. Both the player and NPCs have one, and create a list of entities that they are 'Aware of' via this controller.

AI

The ArtificialIntelligenceController is the top-level script for entity AI. It is generalized across all entities.
