Detailed Description

Tutorial: https://www.youtube.com/watch?v=S9Jd6Ss8tU0

Base class for NPC ability to percieve it's environment. When interacting with parameters from the SensorController, use the SensorInterface.

Attached to: Player/SensorController, or NPC/SensorController, etc.

Inter-Node Communication:

  1. area_entered->_on_Sight_area_entered
  2. area_exited->_on_Sight_area_exited

Programmers Notes:

  1. Signals are connected in _Ready(). Note that contact_monitor must be true and contacts_reported greater than 0.
  2. WARNING: Area2D's need to be directly childed to KinematicBody2D's, or else they do not move with the NPC/Player.
Parameters
_allDetectedEntitiesA list of all detected entities.
_enemiesA list of detected enemies (subset of enemiesAndAllies).
_alliesA list of detected allies (subset of enemiesAndAllies).
_debugBoolean flag that determines whether the sensor prints debugging information to the console.
_sensorInterfaceThe interface for the SensorController. Values from the SensorController are pushed to the Interface.
_entityInterfaceThe interface for this Entity. Useful for accessing values like this entities faction, etc.
_childRaycastFrontReference to the right-facing Raycast2D attached to this Entity.
_childRaycastBackReference to the left-facing Raycast2D attached to this Entity.
_childRaycastFrontCollisionThe object that Raycast2DFront is detecting (null if nothing detected).
_childRaycastBackCollisionThe object that Raycast2DRear is detecting (null if nothing detected).
_frontRaycast2DFlagIs True when something is detected by the front-facing Raycast2DFront node, otherwise false.
_backRaycast2DFlagIs True when something is detected by the rear-facing Raycast2DRear node, otherwise false.
myFactionThe faction of the Entity that the SensorController is attached to.
