Detailed Description

For a tutorial on how these controls were implemented, check out: https://www.youtube.com/watch?v=TrX6WCu5VNc

Attached in scene to:

  1. Player/Scripts/Controls

Inter-Node Communication:

  1. Uses GetParent().GetParent<KinematicBody2D>() to access the KinematicBody2D within the Player node.

Programmers Notes:

  1. MoveAndSlide() and IsOnFloor() work together. IsOnFloor() is perpetually false, unless you specify a floor direction in MoveAndSlide(), i.e. MoveAndSlide(Vector2.UP)
  2. Directly references the input manager.
  3. The player is a flying creature, so it's gravity is set to 0.
Parameters
velocityThe 2-Dimensional velocity (X', Y') of the player, which is directly modified using directional controls.
playerKinematic2DStores a reference to the KinematicBody2D node of the player.
playerSpriteStores a reference to the Sprite node of the player.
SPEEDConstant scalar value that determines the magnitude of the players left/right velocity.
GRAVITYConstant scalar value that determines the rate at which the player 'Falls' in the +y direction. Note that Godot default 'Down' is +y, and 'Up' is -y.
JUMPFORCEConstant scalar value that determines the magnitude of the players 'Up' movement when pressing the 'ui_up' input. Note that this input can only be applied when the player is currently on a 'Floor'.
