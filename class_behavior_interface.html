Detailed Description

Tutorial: N/A

BehaviorInterface stores parameters pertinent to different behaviors (such as patrol waypoints), as well as the currently active behavior. The BehaviorController pulls the currently active behavior from this interface. If the currently active behavior has changed since the last cycle, it will change state to this new behavior and set it active. The Behaviors themselves are child classes of BehaviorTemplate, and utilize the parameters stored in this class. To change a particular behaviors inputs (such as patrol waypoints), use this interface.

Attached to: NPC/Controllers

Inter-Node Communication:

  1. Works with BehaviorController to drive state changes (the controller examines this interface in order to know which state it should be in).
  2. Works with the ArtificialIntelligenceController to drive logic (the controller sets the currentState).
