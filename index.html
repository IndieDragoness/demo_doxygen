Introduction

This is the documentation for the Kobold Tribes project.

Project Notes

At this point in time, the pattern for this project is as follows:

  1. Uses C#: Scripts control player movement, etc. and C# is the base language for the project.
  2. Parameters: Parameters come in two flavors, globals and locals. Locals are specific to a script, and usually private. Globals are stored in Global Parameter scripts such as PlayerGlobals.cs in the case of the player. All globals fall under the scene node hierarchy of GLOBALS/<type_of_global> to reduce inefficient searching. Useful for the User Interface as well.
  3. Character Sprites: All character sprites are either left-facing or right-facing. This means sprites and animations need only be animated in a single direction, which can then be scale-reversed (applying -1 to the scale to flip the horizontal direction) to cover both left/right directions.
  4. Pathfinding: As simple as left or right. Any time an NPC needs to move 'Up' or 'Down' it will encounter this decision while moving left or right and perform logic to make the determination directly above or below the change in plane.
  5. Scripts Referencing each Other and Nodes: The following link explains inter-node communication. This project is organized in such a way that scripts are organized into a hierarchy of Script-themed Nodes that attach to an entity, such as the Player, or objects.
